# Kubernetes Core Concepts Documentation

Welcome to the Kubernetes Core Concepts documentation. This guide provides an overview of essential Kubernetes components and functionalities, helping you understand the architecture, services, namespaces, and command usage within a Kubernetes environment.

## Table of Contents

- [Architecture](architecture.md): Learn about the key components of a Kubernetes cluster, including ETCD, API server, kube scheduler, kube control manager, kube proxy, kubelet, and pods. This section includes detailed explanations and example YAML configurations for pods.

- [Services](services.md): Explore the different types of services in Kubernetes, such as Cluster IP, Load Balancer, and NodePort services. This section explains the purpose of each service type and provides example YAML configurations.

- [Namespaces](namespaces.md): Understand the concept of namespaces in Kubernetes, their purpose, and how they can be used to organize resources. This section includes an example YAML configuration for creating a namespace.

- [Commands](commands.md): Discover the difference between imperative and declarative commands in Kubernetes. This section explains the use of the `kubectl apply` command and provides examples of both command styles.

- [Examples](examples/): Find example YAML configurations for various Kubernetes resources, including pods, services, and namespaces.

## Getting Started

To navigate through the documentation, refer to the links in the Table of Contents. Each section provides in-depth information and practical examples to help you effectively utilize Kubernetes in your projects.

For further assistance or inquiries, please refer to the official Kubernetes documentation or community resources. Happy learning!
